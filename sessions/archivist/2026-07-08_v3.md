## 5. [knowledge] Wikipedia: Ontology (Information Science) — Domain Knowledge for the Anne Project

**Source:** Ontology (information science), Wikipedia

**Directly applicable to the equipment data model design task.**

| Concept | Definition | Application to Anne's 80-Item Ontology |
|---------|-----------|----------------------------------------|
| **Domain ontology** | Concepts belonging to a specific realm | Home equipment ontology: each item (furnace, water heater, filter) is a class with properties and relations |
| **Individuals (instances)** | Specific objects in the ontology | "Kitchen sink in Unit 2" is an instance of the Sink class |
| **Classes** | Categories of entities | 13 categories: kitchen, laundry, HVAC, water, garage, security, entertainment, outdoor, fireplace, backup power, smart home, bathroom, common-forgotten built-ins |
| **Attributes** | Properties that instances carry | Make, model, serial number, purchase date, installation date, warranty expiration, maintenance history |
| **Relations** | How entities connect | "Has-part" (water heater → pressure relief valve), "Contains" (house → room → item), "Attached" (manual → equipment) |
| **Upper ontology** | Shared foundational categories | Standard entity types (Person, Organization, Document, Location, Equipment, Service) |
| **Ontology learning** | Semi-automatic creation from text | The 13-section docx can seed the ontology via information extraction |
| **Ontology editors** | Tools for building | Protégé is the reference; our data model spec achieves the same via structured JSON/Ontology language |

**Key design insight from Gruber (1993):** "Ontologies need not be limited to taxonomic hierarchies of classes — they must include axioms that constrain interpretations." For Anne's app: the ontology must encode rules, not just categories. Example: "If appliance.warranty_expiration < today, display renewal prompt" is an axiom — not a property.

**Recommendation for the probe spec:** The first delegation to Opus should include the upper ontology foundation (Person, Location, Equipment, Document, Service, Task) and the home-domain classes, with at least 5 relation types (has-installation-location, has-manufacturer, has-part, has-maintenance-history, has-manual). This gives Opus the foundational structure to generate the complete ontology.
